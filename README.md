# 日常任务打卡系统

一个简洁实用的Web版日常任务打卡应用,帮助你追踪和坚持每日习惯。

## ✨ 功能特性

- 📝 **任务管理**: 自由添加、编辑、删除打卡项目
- ✅ **打卡确认**: 点击任务后二次确认,防止误操作
- 🎨 **视觉反馈**: 打卡成功后显示对勾图标和绿色高亮
- 📅 **实时日历**: 直观展示打卡记录,支持月份切换
- 📊 **数据统计**:
  - 连续打卡天数
  - 最长打卡记录
  - 月度统计和完成率
  - 30天打卡趋势图
- 💾 **数据备份**: 支持导出/导入JSON文件,数据安全可控
- 🔄 **自动重置**: 每个自然日自动重置打卡状态

## 🚀 快速开始

### 方式一: 本地使用

1. 下载本项目文件夹
2. 双击 `index.html` 文件
3. 在浏览器中打开即可使用

### 方式二: 在线访问

将项目部署到静态托管服务(如 GitHub Pages),通过链接访问。

#### 部署到 GitHub Pages 步骤:

1. 创建一个新的 GitHub 仓库
2. 将所有文件上传到仓库
3. 进入仓库 Settings → Pages
4. 选择分支(通常是 main/master)和根目录
5. 保存后即可获得访问链接

## 📖 使用说明

### 添加任务

1. 点击"添加任务"按钮
2. 输入任务名称
3. 可选择任务颜色(用于区分不同任务)
4. 点击"确认"完成添加

### 打卡

1. 点击任务卡片
2. 在弹出的确认对话框中点击"确定"
3. 打卡成功后,任务卡片会变为绿色并显示✓标记

### 编辑任务

- **修改名称**: 双击任务名称,编辑后按Enter或失去焦点自动保存
- **删除任务**: 鼠标悬停任务卡片,点击右上角的×按钮

### 查看日历

- 点击"上月"/"下月"按钮切换月份
- 点击"回到今天"快速返回当前月份
- 有打卡记录的日期会显示绿色圆点
- 点击任意日期可查看该日打卡详情

### 查看统计

- **总体统计**: 显示总任务数、今日完成情况、累计打卡次数
- **任务坚持情况**: 每个任务的连续天数、最长记录、累计天数
- **本月统计**: 当月打卡次数、活跃天数、完成率
- **趋势图表**: 近30天打卡频率可视化

### 数据备份

- **导出数据**: 点击"导出数据"按钮,下载JSON备份文件
- **导入数据**: 点击"导入数据"按钮,选择之前导出的JSON文件

⚠️ **注意**: 导入数据会覆盖当前所有数据,请谨慎操作!

## 💡 使用技巧

1. **定期备份**: 建议每周或每月导出一次数据,防止浏览器数据丢失
2. **颜色分类**: 为不同类型的任务设置不同颜色,便于区分
3. **查看历史**: 通过日历可以快速回顾任意一天的打卡情况
4. **保持连续**: 关注连续打卡天数,培养坚持的习惯

## 🔒 隐私说明

- 所有数据存储在浏览器本地(LocalStorage)
- 不会上传任何数据到服务器
- 数据完全由用户控制

## 🐛 常见问题

**Q: 清除浏览器缓存会丢失数据吗?**
A: 是的,LocalStorage数据会被清除。请定期使用"导出数据"功能备份。

**Q: 可以在手机上使用吗?**
A: 可以,界面已做响应式设计,支持移动端浏览器访问。

**Q: 如何分享给朋友使用?**
A: 可以将整个文件夹打包发送,或部署到GitHub Pages等静态托管服务生成在线链接。

**Q: 一天可以多次打卡同一个任务吗?**
A: 不可以,每个任务每天只能打卡一次,以自然日为界限。
